AUPRC 只盯着正样本看——这不是傲慢，是诚实

2026-05-08 22:36 80 min read -- 人阅读

从不平衡数据的诚实镜子，到精确率与召回率的权衡艺术，一份覆盖原理、陷阱与代码的深度指南

一、AUROC 给了你一个漂亮的数字，然后呢？

先回顾一个场景。你有 10000 个样本，其中 100 个是正样本（1%）。你训练了一个模型，AUROC = 0.92。论文里这个数字足够好看了，你甚至觉得可以交差了。

但你仔细看了一下模型的输出：它把所有正样本排进了前 500 名（前 5%）。这个排序能力在 AUROC 看来确实不错——随机抽一个正样本和一个负样本，正样本排在前面的概率是 92%。可是在实际业务中，前 500 名意味着 400 个假阳性。你的客服团队需要给这 500 个人逐一打电话确认，其中 80% 是白费力气。

AUROC 不关心这个。AUPRC 关心。

AUPRC（Area Under the Precision-Recall Curve，精确率-召回率曲线下面积）衡量的是：在所有可能的阈值下，模型在找到正样本（召回率）的同时，保持了多高的命中率（精确率）。它专门盯着正样本看，对那 9900 个负样本不屑一顾——这不是傲慢，是在不平衡数据中唯一的诚实态度。

二、精确率和召回率：两个互相掐架的指标

在讲 AUPRC 之前，必须先把 Precision 和 Recall 搞透。它们俩是一对经典的冤家。

**精确率（Precision）**回答的问题是："模型说'是'的样本里，有多少真的是？"

Precision=TPTP+FP\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}} Precision=TP+FPTP​

**召回率（Recall）**回答的问题是："所有真的正样本里，模型找到了多少？"

Recall=TPTP+FN\text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}} Recall=TP+FNTP​

它们为什么要掐架？

因为阈值决定了 TP、FP、FN 的分布，而 Precision 和 Recall 对阈值变化的反应方向是相反的。

阈值很低时（比如 0.1），模型大手一挥把大部分样本判为正。FN 很少（漏掉的少）→ Recall 很高。但 FP 暴多（误报的多）→ Precision 很低。

阈值很高时（比如 0.9），模型谨慎至极，只把最有把握的样本判为正。FP 很少 → Precision 很高。但 FN 很多（漏掉的多）→ Recall 很低。

这就造成了一个经典的权衡：你想要高召回率（宁可错杀一千也不放过一个），就得接受低精确率。你想高精确率（不出手则已，出手必中），就得接受低召回率。

PR 曲线就是描绘这个权衡的完整图谱。

一个具体的数值走查，帮你彻底搞懂阈值的作用。 假设你有 10 个样本，模型给出的预测分数和真实标签如下：

样本 预测分数 真实标签
A 0.95 正
B 0.91 正
C 0.78 负
D 0.72 正
E 0.65 负
F 0.51 负
G 0.43 正
H 0.33 负
I 0.12 正
J 0.05 负

正样本：5 个（A, B, D, G, I），负样本：5 个（C, E, F, H, J）。

现在分别用三个不同的阈值把分数变成标签，看看 Precision 和 Recall 怎么变化：

阈值 = 0.30（低阈值，激进策略）：

预测为正的样本：A, B, C, D, E, F, G, H（分数 ≥ 0.30 的共 8 个）。

其中真的正样本：A, B, D, G（4 个 TP）。假的：C, E, F, H（4 个 FP）。漏掉的：I（1 个 FN，因为分数 0.12 < 0.30）。

  • Precision = 4 / (4 + 4) = 0.50
  • Recall = 4 / (4 + 1) = 0.80

阈值 = 0.70（中阈值，平衡策略）：

预测为正的样本：A, B, C, D（分数 ≥ 0.70 的共 4 个）。

其中真的正样本：A, B, D（3 个 TP）。假的：C（1 个 FP）。漏掉的：G, I（2 个 FN）。

  • Precision = 3 / (3 + 1) = 0.75
  • Recall = 3 / (3 + 2) = 0.60

阈值 = 0.90（高阈值，保守策略）：

预测为正的样本：A, B（分数 ≥ 0.90 的共 2 个）。

其中真的正样本：A, B（2 个 TP）。假的：0 个 FP。漏掉的：D, G, I（3 个 FN）。

  • Precision = 2 / (2 + 0) = 1.00
  • Recall = 2 / (2 + 3) = 0.40

把这三个点画在 (Recall, Precision) 坐标平面上：

阈值 Recall (X) Precision (Y)
0.30（激进） 0.80 0.50
0.70（平衡） 0.60 0.75
0.90（保守） 0.40 1.00

阈值越低 → Recall 越高（X 轴向右）但 Precision 越低（Y 轴向下）。阈值越高 → Precision 越高（Y 轴向上）但 Recall 越低（X 轴向左）。这三个点在坐标平面上构成了一条从左下到右上的下降曲线——这就是 PR 曲线的雏形。

如果你把所有可能的阈值（不仅仅是 0.3 / 0.7 / 0.9）都算一遍，(Recall, Precision) 点会连成一条完整的曲线。而曲线下的面积，就是 AUPRC。

三、PR 曲线的几何直觉

PR 曲线的构建逻辑比 ROC 曲线更直接——它使用 Precision 和 Recall 这两个你已经熟悉的指标。

对于每一个可能的阈值，计算一对 (Recall, Precision)，然后在以 Recall 为 X 轴、Precision 为 Y 轴的坐标平面上描点。阈值从低到高遍历完，这些点连成一条曲线，就是 PR 曲线。

但这里有一个重要的细节：PR 曲线不是单调的。与 ROC 曲线不同，PR 曲线在阈值连续变化时可以上下抖动。这是因为 Precision 不仅取决于 TP，还取决于 FP——当阈值变化时，FP 的变化可能突然增大或减小，导致 Precision 出现非连续的跳跃。

举个例子：假设相邻两个样本的分数分别是 0.92 和 0.89。当阈值从 0.93 降到 0.91 时，分数为 0.92 的那个样本被纳入"预测为正"的集合。如果它是一个真阳性（TP），Precision 会上升或保持不变；如果它是一个假阳性（FP），Precision 会骤降。这就是 PR 曲线锯齿状的来源。

sklearn 的 precision_recall_curve 函数会自动对曲线进行插值平滑处理，保证计算出的 AUPRC 是一个稳定的值，不受锯齿影响。

AUPRC 就是这条曲线下的面积。

from sklearn.metrics import precision_recall_curve
import numpy as np

# 模拟数据：10 个样本，5 个正，5 个负
y_demo = np.array([1, 1, 0, 1, 0, 0, 1, 0, 1, 0])
scores_demo = np.array([0.95, 0.91, 0.78, 0.72, 0.65, 0.51, 0.43, 0.33, 0.12, 0.05])

precision, recall, thresholds = precision_recall_curve(y_demo, scores_demo)
print(f"precision shape: {precision.shape}")   # (11,) — 比阈值多 1
print(f"recall shape:    {recall.shape}")      # (11,)
print(f"thresholds shape:{thresholds.shape}")  # (10,) — 最后一个点无对应阈值
print(f"\n最后一个点: recall={recall[-1]:.2f}, precision={precision[-1]:.2f}")
print("（对应阈值最低、全部判正时的极端情况）")

precision shape: (11,)
recall shape:    (11,)
thresholds shape:(10,)

最后一个点: recall=0.00, precision=1.00
（对应阈值最低、全部判正时的极端情况）

PR 曲线的一个重要特征：它从右上角的 (1, 正样本比例) 附近开始（当阈值极低、全部判正时），向左上方逐渐移动（随着阈值升高、精确率提升、召回率下降），最终落到某个位置。

一个完美的模型，PR 曲线会紧贴右上角——在所有召回率水平上精确率都接近 1.0。AUPRC ≈ 1.0。

一个随机模型，PR 曲线是一条水平线，高度等于正样本比例。如果 1% 是正样本，随机模型的 AUPRC ≈ 0.01。这和 AUROC 的基线固定为 0.5 截然不同——AUPRC 的基线由数据本身的分布决定。

一个灾难模型（把所有正样本排到所有负样本之后），PR 曲线会掉到基线以下，AUPRC < 基线。

四、AUPRC 的基线为什么不是固定的？

这是理解 AUPRC 最关键也最容易被忽略的一点。

AUROC 的基线永远是 0.5——因为即使是随机抛硬币，正样本排到负样本前面的概率也是 50%。

AUPRC 的基线是正样本比例。如果你的数据中正样本占 10%，一个随机模型的 AUPRC ≈ 0.10。如果正样本只占 1%，基线 ≈ 0.01。

数学推导：

对于一个随机猜测模型（模型输出的分数和真实标签独立），在任意阈值下，模型预测为"正"的样本中，真正是正样本的比例等于正样本在全体数据中的比例。设正样本比例为 ppp，则对任意阈值：

Precision≈p（对所有召回率水平） \text{Precision} \approx p \quad \text{（对所有召回率水平）} Precision≈p（对所有召回率水平）

因此 PR 曲线近似为一条水平线 y=py = py=p，曲线下面积为：

AUPRCrandom≈p×1=p\text{AUPRC}_{\text{random}} \approx p \times 1 = p AUPRCrandom​≈p×1=p

为什么这个性质如此重要？

因为 AUROC 的基线固定在 0.5，不管你数据是什么样。这意味着一个 AUROC = 0.7 的模型，在正样本 50% 的数据上和在正样本 1% 的数据上，看起来都"比随机好 0.2"。但实际上，在后一种情况下，模型克服了巨大的不平衡困难才达到这个水平，付出的努力和模型能力远超前一种情况。

而 AUPRC 通过让基线随数据变化，反映了这种困难。正样本 50% 时基线 0.5 → AUPRC 做到 0.7 只提升了 0.2。正样本 1% 时基线 0.01 → AUPRC 做到 0.7 提升了 0.69，这需要极强的区分能力。

这就是为什么在极不平衡数据上，AUPRC 是比 AUROC 更诚实的指标。

import numpy as np

# 模拟 1000 个样本，2% 正样本（20 个）
y_demo = np.zeros(1000)
y_demo[:20] = 1

baseline = y_demo.sum() / len(y_demo)
print(f"正样本比例（基线）: {baseline:.4f}")
print(f"随机模型 AUPRC ≈ {baseline:.4f}（因为 Precision ≈ 正样本比例）")
print(f"对比：AUROC 的基线固定是 0.5，不受此影响")

五、AUPRC vs AUROC：什么时候该用哪个？

维度 AUROC AUPRC
关注对象 正负样本整体的排序质量 正样本的排序质量和命中率
基线 固定 0.5 正样本比例（随数据变化）
对类别不平衡的敏感度 低（理论层面对此鲁棒） 高（设计层面就关注稀有类）
对 FPR 的依赖 直接（FPR 是 X 轴） 间接（FP 通过 Precision 体现）
对 TN 的依赖 直接（TN 在 FPR 的分母中） 零（公式中完全不出现 TN）
可跨数据集比较 可以 不可以（基线不同）

一个简单的决策规则：

  • 正样本比例 ≥ 20%，正负样本大致均衡 → AUROC 和 AUPRC 都可以，AUROC 更通用
  • 正样本比例 5% – 20% → 两者同时报告，重点看 AUPRC
  • 正样本比例 < 5%，或者你的业务目标聚焦在正样本上（罕见病、欺诈、高危客户）→ 必须用 AUPRC，AUROC 可能给出虚假的安全感

六、为什么 AUPRC 对极不平衡数据更诚实：深度剖析

这一节给那些不满足于"AUPRC 更好"、想理解为什么更好的人。

FPR 的欺骗性。

AUROC 的 X 轴是 FPR（假阳性率），定义为 FPFP+TN\frac{\text{FP}}{\text{FP} + \text{TN}}FP+TNFP​。在极不平衡数据中（比如正样本 1%），TN 的数量极其庞大。这意味着即使 FP 很多，FPR 看起来也很小。

举个例子：10000 个样本中 100 个正样本（1%），9900 个负样本。模型把正样本排得稀烂，导致了 990 个假阳性（FP）。但 FPR = 990 / 9900 = 0.10。ROC 曲线上的这个点（FPR=0.10, TPR=某个值）看起来并不可怕——毕竟 FPR 才 10%。

但 PR 曲线用 Precision 来看同样的事：模型预测了 100 + 990 = 1090 个"正样本"，其中只有 100 个是真的。Precision = 100 / 1090 ≈ 0.092。这个数字直接告诉你：模型说"正"的时候，十次里有九次是错的。

一个是 10% 的"假阳性率"——听起来还行。一个是 9.2% 的"命中率"——听起来完全不行。它们描述的是同一个模型，但 PR 视角揭露了 AUROC 视角掩盖的真相。

TN 的削弱效应。

AUROC 的 FPR 公式中，分母是 FP + TN。在极不平衡数据中，TN 是一个巨大的缓冲垫，吸收了 FP 的影响。FP 增加 1000，FPR 从 0.01 变成 0.11——只变了 0.10。

但在 PR 曲线中，Precision 的分母是 TP + FP。这里没有 TN。FP 增加 1000，分母暴增，Precision 从 0.5 掉到 0.09——直接腰斩再腰斩。

这就是为什么 AUROC 可以"假装"模型还不错，而 AUPRC 不会。

一个残酷的对比实验。

在你的脑子里做这个实验：10000 个样本，50 个正样本（0.5%）。一个模型给出以下结果：

  • 完美的正样本排序：所有 50 个正样本排在前 500 名（前 5%）
  • FP 数量：450 个（前 500 名中，除去 50 个正样本，剩下 450 个全是负样本）
  • FN 数量：0（全部找到）

Precision = 50 / 500 = 0.10。Recall = 50 / 50 = 1.0。
FPR = 450 / 9950 ≈ 0.045。TPR = 1.0。

AUROC ≈ 0.978（几乎完美！因为 TPR=1.0 而 FPR 只有 0.045，曲线紧贴左上角）。

AUPRC ≈ 需要计算……但你会看到 Precision 无法在所有召回率水平上都保持在高位——它从 1.0 附近开始，迅速下降到 0.1 附近，曲线大多数区域都很低。

同一个模型，AUROC 说它"近乎完美"，AUPRC 说它"勉强可用"。这不是哪个指标更准确的问题——它们衡量的是不同的东西。AUROC 告诉你正负样本的排序质量（确实很好），AUPRC 告诉你找到正样本的命中率（并不高）。如果你的业务关心的是找到正样本的成本效益，AUPRC 才是对的指标。

七、AUPRC 的陷阱

AUPRC 不是完美的。了解它的软肋才能用好它。

陷阱一：不能跨数据集比较。 如前所述，不同正样本比例的数据集上 AUPRC 基线不同。例如数据集 A 的正样本比例为 50%，AUPRC 基线为 0.50，模型 A 的 AUPRC = 0.85；数据集 B 的正样本比例为 1%，AUPRC 基线为 0.01，模型 B 的 AUPRC = 0.70。你不能说"模型 A 比模型 B 好"，因为模型 A 只比基线高了 0.35，而模型 B 比基线高了 0.69——在克服不平衡难度上，模型 B 远强于 A。如果你必须跨数据集比较，可以报告"AUPRC / 基线"的倍数，或者使用标准化后的指标 AUPRC−基线1−基线\frac{\text{AUPRC} - \text{基线}}{1 - \text{基线}}1−基线AUPRC−基线​。

陷阱二：曲线尾部噪声大。 PR 曲线的最后一段（高召回率区域，对应低阈值）涉及的样本数极少。举个例子：当阈值为 0.05 时，模型可能只把样本中分数最高的 0.3% 判为负，其余全部判正——此时 Precision 的计算分母巨大（几乎全判正），分子是正样本总数。任何一个样本的微小分数变动都可能导致 Precision 剧烈抖动。因此不同模型 AUPRC 的差异主要来自曲线中段（阈值 0.2-0.8 区域），但简单的面积计算对尾部和中段一视同仁。如果你怀疑尾部噪声在误导比较，使用 Partial AUPRC（只计算特定 Recall 区间的面积）。

陷阱三：完美排序不等于高 AUPRC，反之亦然。 两个模型可以有相同的 AUPRC 但完全不同的 Precision-Recall 特性。模型 X 在所有正样本排到前 10% 时精确率极高（0.95），但后 90% 的召回率区域精确率暴跌（0.02）——它的 AUPRC 可能和模型 Y 一样，但模型 Y 的精确率在所有召回率水平上都稳定在 0.5 左右。只看 AUPRC 数字，你无法区分这两种模型。永远不要只看 AUPRC 数字而不看 PR 曲线本身。

陷阱四：不能替代业务指标。 以一个反欺诈场景为例：每天 100 万笔交易，其中 100 笔是欺诈（0.01%）。你的风控团队每天只能人工审核 500 笔。你需要的不是全阈值范围内的 AUPRC，而是 Top-500 预测中的精确率（Precision@500）。即使模型 C 的 AUPRC 高于模型 D，如果模型 D 的 Precision@500 更高，你就应该选 D。AUPRC 是很好的离线评估指标，但业务相关的局部指标才是上线决策的最终依据。

八、AUPRC 的变体：哪个更适合你？

AUPRC 在实际使用中有几个变体，每个适合不同场景：

Average Precision (AP)： sklearn.metrics.average_precision_score。在 PR 曲线上用插值法计算的面积。这是最常用的 AUPRC 实现，也是 sklearn 中的标准。

Precision@K： 不看全局，只看模型给出的 Top-K 个预测中精确率是多少。适合推荐系统、信息检索等"只关心前 K 个结果"的场景。

R-Precision： 取 K = 真实正样本数，计算 Top-K 的精确率。结合了数据本身的规模和模型的排序能力。

F1 Score（及 Fβ）： 不是面积指标，而是 Precision 和 Recall 在某个特定阈值下的调和平均。Fβ 中的 β < 1 偏重 Precision（比如反欺诈需要精确率），β > 1 偏重 Recall（比如癌症筛查需要召回率）。

from sklearn.metrics import average_precision_score, f1_score
import numpy as np

# 模拟数据（5 个样本）
y_demo = np.array([1, 0, 1, 0, 1])
scores_demo = np.array([0.9, 0.8, 0.7, 0.4, 0.3])

# AUPRC = Average Precision
ap = average_precision_score(y_demo, scores_demo)

# 特定阈值下的 F1
y_pred = (scores_demo >= 0.5).astype(int)
f1 = f1_score(y_demo, y_pred)

print(f"AUPRC (AP): {ap:.4f}")
print(f"F1@0.5:    {f1:.4f}")

AUPRC (AP): 0.7556
F1@0.5:    0.6667

九、代码实战

9.1 基础 AUPRC 计算与 PR 曲线绘制

这段代码做了四件事：生成极不平衡的模拟数据（2% 正样本）、训练逻辑回归、计算 AUPRC 和 AUROC、打印对比。注意我们用 weights=[0.98, 0.02] 制造了严重的不平衡——这正是 AUPRC 应该登场的数据场景。

import numpy as np
import matplotlib.pyplot as plt
from sklearn.metrics import (
    precision_recall_curve, average_precision_score,
    roc_auc_score
)
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression

# 设置中文显示
plt.rcParams['font.sans-serif'] = ['Arial Unicode MS', 'PingFang SC', 'SimHei', 'STHeiti']
plt.rcParams['axes.unicode_minus'] = False

np.random.seed(42)

# 生成极度不平衡数据（2% 正样本）
X, y = make_classification(
    n_samples=5000, n_features=20, n_informative=10,
    weights=[0.98, 0.02], random_state=42
)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)
y_scores = model.predict_proba(X_test)[:, 1]

# 计算指标
auprc = average_precision_score(y_test, y_scores)
auc_roc = roc_auc_score(y_test, y_scores)
baseline = y_test.sum() / len(y_test)

print(f"正样本比例（基线）: {baseline:.4f}")
print(f"AUROC: {auc_roc:.4f}")
print(f"AUPRC: {auprc:.4f}")
print(f"AUPRC / 基线: {auprc / baseline:.2f} 倍")

正样本比例（基线）: 0.0220
AUROC: 0.8491
AUPRC: 0.5342
AUPRC / 基线: 24.28 倍

9.2 绘制 PR 曲线

这段代码画出 PR 曲线并标注了三个关键阈值点。注意我们特别画了一条水平虚线作为随机基线——正样本比例只有 2%，所以基线在 Y=0.02 附近。一条好的 PR 曲线会明显高于这条线，而且越贴近顶部越好。

precision, recall, thresholds = precision_recall_curve(y_test, y_scores)

fig, ax = plt.subplots(figsize=(8, 7))
ax.plot(recall, precision, 'r-', lw=2.5, label=f'模型 (AUPRC = {auprc:.4f})')
ax.axhline(y=baseline, color='k', linestyle='--', lw=1.5, alpha=0.6,
           label=f'随机基线 = {baseline:.4f}')
ax.fill_between(recall, precision, alpha=0.12, color='red')

ax.set_xlabel('召回率 (Recall)', fontsize=13)
ax.set_ylabel('精确率 (Precision)', fontsize=13)
ax.set_title('Precision-Recall 曲线\n（基线随正样本比例变化，不是 0.5）', fontsize=14)
ax.legend(fontsize=12)
ax.set_xlim([0, 1.02])
ax.set_ylim([0, 1.02])
ax.grid(True, alpha=0.3)

# 标注关键阈值
for thresh in [0.3, 0.5, 0.7]:
    idx = np.argmin(np.abs(thresholds - thresh))
    ax.plot(recall[idx], precision[idx], 'o', color='darkred', markersize=7)
    ax.annotate(f'阈值={thresh}', (recall[idx], precision[idx]),
                textcoords="offset points", xytext=(10, -10), fontsize=9)

plt.tight_layout()
plt.show()

9.3 PR 曲线 vs ROC 曲线：不平衡数据的残酷对比

这是本文最关键的图。左边 ROC、右边 PR，同一个模型、同一份数据（2% 正样本）。观察 ROC 曲线——基线 0.5，曲线轻松超越，看起来模型不错。再看 PR 曲线——基线只有 0.02，曲线在极低的高度上挣扎。这张对比图胜过千言。

from sklearn.metrics import roc_curve

fpr, tpr, _ = roc_curve(y_test, y_scores)

fig, axes = plt.subplots(1, 2, figsize=(15, 6))

# 左：ROC
axes[0].plot(fpr, tpr, 'b-', lw=2.5, label=f'AUROC = {auc_roc:.4f}')
axes[0].plot([0, 1], [0, 1], 'k--', lw=1.5, alpha=0.5, label='随机基线 = 0.5')
axes[0].fill_between(fpr, tpr, alpha=0.1, color='blue')
axes[0].set_xlabel('假阳性率 (FPR)', fontsize=12)
axes[0].set_ylabel('真阳性率 (TPR)', fontsize=12)
axes[0].set_title(f'ROC 曲线\n（正样本仅 {baseline:.0%}，基线仍是 0.5）', fontsize=14)
axes[0].legend(fontsize=11)
axes[0].grid(True, alpha=0.3)

# 右：PR
axes[1].plot(recall, precision, 'r-', lw=2.5, label=f'AUPRC = {auprc:.4f}')
axes[1].axhline(y=baseline, color='k', linestyle='--', lw=1.5, alpha=0.5,
                label=f'随机基线 = {baseline:.4f}')
axes[1].fill_between(recall, precision, alpha=0.1, color='red')
axes[1].set_xlabel('召回率 (Recall)', fontsize=12)
axes[1].set_ylabel('精确率 (Precision)', fontsize=12)
axes[1].set_title(f'PR 曲线\n（基线只有 {baseline:.4f}，诚实地反映了稀有性）', fontsize=14)
axes[1].legend(fontsize=11)
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

这张图是整篇文章最核心的可视化。ROC 曲线在 98% 负样本的情况下依然光鲜——基线永远是 0.5，曲线轻轻松松就能抬头。PR 曲线的基线却只有 0.02，要在这之上做出大幅提升，模型必须有真正的区分能力。

9.4 不同正样本比例下的 AUPRC 基线变化

这段代码展示了同一个理想模型（正样本全部排在最前面）在不同正样本比例下的 PR 曲线。你看到的不是模型变差了——而是基线在随正样本比例剧烈移动。五条曲线的形状完全一样，但高度天差地别，因为基线的锚点变了。这直观地说明了为什么不能跨数据集比较绝对 AUPRC 值。

ratios = [0.5, 0.2, 0.1, 0.05, 0.02, 0.01]

fig, ax = plt.subplots(figsize=(9, 6))

for ratio in ratios:
    # 模拟该比例下的理想模型（所有正样本排在最前面）
    n_pos = int(1000 * ratio)
    n_neg = 1000 - n_pos
    y_true_sim = np.concatenate([np.ones(n_pos), np.zeros(n_neg)])
    # 理想模型：正样本分数全部为 1，负样本全部为 0
    y_scores_sim = np.concatenate([np.ones(n_pos), np.zeros(n_neg)])

    prec, rec, _ = precision_recall_curve(y_true_sim, y_scores_sim)
    ap = average_precision_score(y_true_sim, y_scores_sim)
    ax.plot(rec, prec, lw=2, alpha=0.8, label=f'正样本 {ratio:.0%}, 基线={ratio:.4f}')

ax.set_xlabel('召回率 (Recall)', fontsize=13)
ax.set_ylabel('精确率 (Precision)', fontsize=13)
ax.set_title('不同正样本比例下的 PR 曲线\n（基线随正样本比例大幅移动）', fontsize=14)
ax.legend(fontsize=11, loc='lower left')
ax.grid(True, alpha=0.3)
ax.set_xlim([0, 1])
ax.set_ylim([0, 1.02])

plt.tight_layout()
plt.show()

9.5 多模型 AUPRC 对比

from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier

models = {
    '逻辑回归': LogisticRegression(max_iter=1000),
    '随机森林': RandomForestClassifier(n_estimators=100, random_state=42),
    '梯度提升': GradientBoostingClassifier(n_estimators=100, random_state=42),
}

fig, ax = plt.subplots(figsize=(9, 7))
colors = ['#E53935', '#43A047', '#1E88E5']

for (name, model), color in zip(models.items(), colors):
    model.fit(X_train, y_train)
    scores = model.predict_proba(X_test)[:, 1] if hasattr(model, 'predict_proba') \
             else model.decision_function(X_test)
    prec, rec, _ = precision_recall_curve(y_test, scores)
    ap = average_precision_score(y_test, scores)
    ax.plot(rec, prec, lw=2.5, color=color, alpha=0.85,
            label=f'{name} (AUPRC = {ap:.4f})')

ax.axhline(y=baseline, color='k', linestyle='--', lw=1.5, alpha=0.5,
           label=f'随机基线 = {baseline:.4f}')
ax.set_xlabel('召回率 (Recall)', fontsize=13)
ax.set_ylabel('精确率 (Precision)', fontsize=13)
ax.set_title('多模型 AUPRC 对比\n（极不平衡数据，2% 正样本）', fontsize=14)
ax.legend(fontsize=12)
ax.grid(True, alpha=0.3)

9.6 AUROC 和 AUPRC 同时报告（顶会论文标准做法）

from sklearn.metrics import classification_report, brier_score_loss

# 阈值 0.5 下的分类报告
y_pred = (y_scores >= 0.5).astype(int)
print("=== 阈值 = 0.5 下的分类报告 ===")
print(classification_report(y_test, y_pred, target_names=['负类', '正类']))

# 综合报告
print("=== 综合指标报告 ===")
print(f"正样本比例:           {baseline:.4f} ({baseline*100:.2f}%)")
print(f"AUROC:                {auc_roc:.4f}")
print(f"AUPRC (AP):           {auprc:.4f}")
print(f"AUPRC 基线:           {baseline:.4f}")
print(f"AUPRC 超基线倍数:     {auprc / baseline:.2f}x")
print(f"Brier Score:          {brier_score_loss(y_test, y_scores):.4f}")

十、总结

AUPRC 是机器学习的诚实指标。它不掩饰类别不平衡的挑战，不给你虚假的安全感，不让你拿一个好看的 AUROC 搪塞过去。它的核心价值是：在不平衡数据中，告诉你模型在寻找稀有正样本这件事上到底有多靠谱。

记住三个原则：

  • 正样本 < 10% 时，AUPRC 必须和 AUROC 一起报告，AUPRC 是主指标
  • 报告 AUPRC 时必须同时报告正样本比例（基线），否则数字不可解读
  • AUPRC 衡量的是排序，不是校准——需要概率估计时加 Brier Score

一个负责任的评估报告至少包含：AUROC、AUPRC、正样本比例、AUPRC 基线、Brier Score。五个数字，不多不少，恰好讲清模型的全部真相。
