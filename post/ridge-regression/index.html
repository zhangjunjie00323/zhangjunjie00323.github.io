岭回归：宁可有偏，不可失控

2026-05-06 22:38 184 min read 人阅读

当线性回归崩溃时——岭回归的救赎

一、从一个尴尬的场景开始

你花了三天时间做特征工程，构造了 200 个特征来描述 100 个样本。你满怀信心地跑了一个线性回归，然后训练集 R² 高达 0.99。你兴奋地拿给同事看，同事说："验证集跑一下？"你跑了。验证集 R² = -0.3。

你遭遇了机器学习中最经典的陷阱之一——过拟合。

这个问题的数学本质是什么？当你用普通最小二乘法（OLS）估计线性回归的参数时，最优解是：

β^OLS=(XTX)−1XTy\hat{\beta}_{\text{OLS}} = (X^T X)^{-1} X^T y β^​OLS​=(XTX)−1XTy

当特征数量接近甚至超过样本数量时，XTXX^T XXTX 接近奇异矩阵——它的某些特征值趋近于零。求逆的时候，这些微小的特征值被翻转为巨大的倒数，导致参数估计的方差爆炸。你的模型不再学习数据的规律，而是在记忆噪声。

岭回归（Ridge Regression）就是为了解决这个问题而生的。它的核心思想简单得令人发指：在 XTXX^T XXTX 的对角线上加一个小小的扰动 λ\lambdaλ，让矩阵求逆变得稳定。

β^Ridge=(XTX+λI)−1XTy\hat{\beta}_{\text{Ridge}} = (X^T X + \lambda I)^{-1} X^T y β^​Ridge​=(XTX+λI)−1XTy

就这么一行改动，改变了整个线性模型的命运。

二、数学原理：从三个视角理解同一件事

岭回归的美妙之处在于，你可以从三个完全不同的角度去理解它，每一种视角都会让你看到不同的风景。

视角一：数值稳定的补救

OLS 的闭式解

β^=(XTX)−1XTy\hat{\beta} = (X^T X)^{-1} X^T y β^​=(XTX)−1XTy

的稳定性取决于 XTXX^T XXTX 的条件数——最大特征值与最小特征值之比。当特征之间高度相关（多重共线性），条件数会暴增，矩阵求逆变得数值不稳定。

加入 λI\lambda IλI 后，XTX+λIX^T X + \lambda IXTX+λI 的最小特征值从 σmin⁡\sigma_{\min}σmin​ 提升到 σmin⁡+λ\sigma_{\min} + \lambdaσmin​+λ，条件数大幅下降。这是一个纯数值计算的视角：岭回归本质上是对病态矩阵的 Tikhonov 正则化。

视角二：约束优化

另一种理解方式是把岭回归看作带约束的最小二乘：

min⁡β∥y−Xβ∥22s.t.∥β∥22≤t\min_{\beta} \| y - X\beta \|_2^2 \quad \text{s.t.} \quad \|\beta\|_2^2 \leq t βmin​∥y−Xβ∥22​s.t.∥β∥22​≤t

等价地，用拉格朗日乘子法写成：

β^Ridge=arg⁡min⁡β{∥y−Xβ∥22+λ∥β∥22}\hat{\beta}_{\text{Ridge}} = \arg\min_{\beta} \left\{ \| y - X\beta \|_2^2 + \lambda \|\beta\|_2^2 \right\} β^​Ridge​=argβmin​{∥y−Xβ∥22​+λ∥β∥22​}

第一项是损失，衡量模型对数据的拟合程度。第二项是惩罚，衡量参数的大小。λ\lambdaλ 控制两者之间的权重——λ=0\lambda = 0λ=0 退化为 OLS，λ→∞\lambda \to \inftyλ→∞ 则所有参数趋近于零。

这里的

∥β∥22=∑j=1pβj2\|\beta\|_2^2 = \sum_{j=1}^{p} \beta_j^2 ∥β∥22​=j=1∑p​βj2​

是 L2 范数的平方。注意它与 Lasso 的 L1 范数

∥β∥1=∑j=1p∣βj∣\|\beta\|_1 = \sum_{j=1}^{p} |\beta_j| ∥β∥1​=j=1∑p​∣βj​∣

的区别——这个区别决定了岭回归不会把参数压缩到精确的零，只是让它们变小。

视角三：贝叶斯后验

如果你从贝叶斯的角度看，OLS 相当于假设参数服从无信息先验（均匀分布），岭回归则相当于假设参数服从均值为零的高斯先验：

β∼N(0,σ2λI)\beta \sim \mathcal{N}(0, \frac{\sigma^2}{\lambda} I) β∼N(0,λσ2​I)

在这个先验下，参数的 MAP（最大后验）估计恰好等于岭回归的闭式解。λ\lambdaλ 越大，先验越集中（越确信参数接近零），后验估计越向零收缩。

三种视角殊途同归，你可以根据对话对象选择最合适的解释——对工程师讲数值稳定，对统计学家讲约束优化，对贝叶斯信徒讲先验分布。

三、λ\lambdaλ 的行为：一个控制了模型生死的参数

理解 λ\lambdaλ 如何影响模型，是掌握岭回归的关键。

当 λ=0\lambda = 0λ=0 时，岭回归退化回 OLS。如果数据中不存在多重共线性且样本充足，这可能是最优的。

当 λ\lambdaλ 很小时（比如 0.001），正则化的效果微乎其微，参数估计和 OLS 几乎一致。模型方差仍然较大。

当 λ\lambdaλ 适中时，我们进入了甜区。各个参数被不同程度地压缩，变量之间的共线性被抵消，模型偏差略有增加但方差大幅下降——总体的泛化误差达到最优。

当 λ\lambdaλ 很大时，所有参数被强行压向零。模型变得极其简单，偏差主导了误差。极端情况下 λ→∞\lambda \to \inftyλ→∞ 时，β^→0\hat{\beta} \to 0β^​→0，模型退化为一条水平线。

一个重要的观察：岭回归不会将任何系数压缩到精确的零。这是因为 L2 惩罚的梯度和参数大小成正比，当参数趋近于零时，梯度也趋于零，无法产生"推过零点"的动力。如果你需要特征选择（让不重要的参数精确为零），应该用 Lasso。

四、与 OLS 和 Lasso 的对比

参数缩减 不缩减 均匀缩减 选择性缩减（部分归零）
特征选择 不支持 不支持 支持

什么时候用岭回归而不是 Lasso？

当你不确定哪些特征应该被丢弃时。如果你的领域知识告诉你"所有特征都可能有关，只是程度不同"，岭回归的均匀压缩是更合理的选择。另一类典型场景是基因表达数据——成千上万个基因，你知道很多基因参与了通路，但不希望粗暴地丢弃任何信号。岭回归保留所有特征，只是让它们的系数更保守。

什么时候用 Lasso 而不是岭回归？

当你需要可解释的稀疏模型时。比如你要向老板汇报"影响销量的三个最关键因素"，Lasso 会自然地帮你挑出那几个特征，岭回归则会给所有特征分配一个很小的非零系数——技术上正确，但汇报起来很尴尬。

弹性网络（Elastic Net）是折中方案，同时使用 L1 和 L2 惩罚，兼具特征选择和组效应的优点，在实际工程中往往是比纯岭回归或纯 Lasso 更好的默认选择。

5.2 构造多重共线性数据

我们先制造一组"不怀好意"的数据——特征高度相关、样本数不多、再加点噪声：

np.random.seed(42)

# 生成 80 个样本、30 个特征，其中只有 5 个真正有信息量
X, y, true_coef = make_regression(
    n_samples=80, n_features=30, n_informative=5,
    noise=15, coef=True, random_state=42
)

# 人为制造多重共线性：让后 10 个特征是前 10 个特征的线性组合
X[:, 20:30] = X[:, 0:10] + np.random.normal(0, 0.1, (80, 10))

print(f"X shape: {X.shape}")
print(f"真实系数（前 10 个）: {true_coef[:10]}")
print(f"条件数（condition number）: {np.linalg.cond(X.T @ X):.1f}")

5.3 OLS vs Ridge：一次公平的较量

# 划分数据
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42
)

# 标准化：岭回归对尺度敏感，必须先标准化
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

# OLS
ols = LinearRegression()
ols.fit(X_train_scaled, y_train)
y_pred_ols = ols.predict(X_test_scaled)

# Ridge（alpha 即 λ）
ridge = Ridge(alpha=1.0)
ridge.fit(X_train_scaled, y_train)
y_pred_ridge = ridge.predict(X_test_scaled)

print(f"OLS  Test R²: {r2_score(y_test, y_pred_ols):.4f}")
print(f"Ridge Test R²: {r2_score(y_test, y_pred_ridge):.4f}")
print(f"\nOLS 系数范围: [{ols.coef_.min():.1f}, {ols.coef_.max():.1f}]")
print(f"Ridge 系数范围: [{ridge.coef_.min():.1f}, {ridge.coef_.max():.1f}]")

你可以反复运行上面的代码（每次改 random_state）。你会观察到：OLS 的测试集 R² 在不同 split 间剧烈波动（方差大），而岭回归稳定得多。这就是正则化降低方差的直接证据。

5.4 λ\lambdaλ 之旅：从 0.0001 到 10000

这是理解岭回归最直观的方式——观察系数如何随正则化强度的增加而变化：

alphas = np.logspace(-4, 4, 100)  # 0.0001 到 10000

coefficients = []
train_scores = []
test_scores = []

for alpha in alphas:
    ridge = Ridge(alpha=alpha)
    ridge.fit(X_train_scaled, y_train)
    
    coefficients.append(ridge.coef_)
    train_scores.append(ridge.score(X_train_scaled, y_train))
    test_scores.append(ridge.score(X_test_scaled, y_test))

coefficients = np.array(coefficients)

# 左图：系数轨迹
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

for i in range(30):
    axes[0].plot(alphas, coefficients[:, i], lw=1, alpha=0.7)
axes[0].set_xscale('log')
axes[0].set_xlabel('λ (Alpha)', fontsize=12)
axes[0].set_ylabel('系数值', fontsize=12)
axes[0].set_title('岭回归系数路径\n（参数随 λ 增大而向零收缩）', fontsize=13)
axes[0].axhline(y=0, color='black', linestyle='--', alpha=0.3)

# 右图：训练/测试 R² 曲线
axes[1].plot(alphas, train_scores, label='训练集 R²', lw=2)
axes[1].plot(alphas, test_scores, label='测试集 R²', lw=2)
axes[1].set_xscale('log')
axes[1].set_xlabel('λ (Alpha)', fontsize=12)
axes[1].set_ylabel('R²', fontsize=12)
axes[1].set_title('偏差-方差权衡\n（训练 R² 下降 = 偏差增大，测试 R² 先升后降 = 甜区）', fontsize=13)
axes[1].legend()
axes[1].axvline(x=alphas[np.argmax(test_scores)], color='red', linestyle='--',
                label=f'最优 α={alphas[np.argmax(test_scores)]:.2f}')
axes[1].legend()

plt.tight_layout()
plt.savefig('ridge_coefficient_path.png', dpi=150, bbox_inches='tight')
plt.show()

左图展示了最经典的"岭回归系数路径图"——所有系数从 OLS 的大幅值（最左边）开始，随着 λ\lambdaλ 增大（向右）逐步向零收缩。那些震荡最剧烈的系数（对应共线性特征组）是最先被压缩的。

右图展示了偏差-方差权衡的经典 U 形曲线。测试集 R² 在某个 α\alphaα 处达到峰值——这就是你要找的最优正则化强度。

5.5 交叉验证选最优 α\alphaα

手动猜 α\alphaα 是不明智的。交叉验证是标准做法：

# GridSearchCV：系统搜索最优 alpha
param_grid = {'alpha': np.logspace(-3, 3, 50)}

ridge_cv = GridSearchCV(
    Ridge(), param_grid,
    cv=5, scoring='neg_mean_squared_error',
    return_train_score=True
)
ridge_cv.fit(X_train_scaled, y_train)

print(f"最优 alpha: {ridge_cv.best_params_['alpha']:.4f}")
print(f"最优 CV MSE: {-ridge_cv.best_score_:.2f}")

# 最终模型
best_ridge = ridge_cv.best_estimator_
y_pred_best = best_ridge.predict(X_test_scaled)
print(f"Test R² (最优 Ridge): {r2_score(y_test, y_pred_best):.4f}")

最优 alpha: 79.0604
最优 CV MSE: 8065.99
Test R² (最优 Ridge): 0.0875

如果你不想用网格搜索，scikit-learn 提供了更高效的 RidgeCV，内置了高效的留一交叉验证（LOOCV），利用广义交叉验证（GCV）的计算技巧，不需要真的训练 n 次模型：

from sklearn.linear_model import RidgeCV

ridge_cv_fast = RidgeCV(alphas=np.logspace(-3, 3, 100), cv=5)
ridge_cv_fast.fit(X_train_scaled, y_train)
print(f"RidgeCV 最优 alpha: {ridge_cv_fast.alpha_:.4f}")

RidgeCV 最优 alpha: 81.1131

5.6 OLS vs Ridge vs Lasso vs ElasticNet：四方会战

models = {
    'OLS': LinearRegression(),
    'Ridge': Ridge(alpha=1.0),
    'Lasso': Lasso(alpha=0.1, max_iter=10000),
    'ElasticNet': ElasticNet(alpha=0.1, l1_ratio=0.5, max_iter=10000)
}

# 5 折交叉验证评估
results = {}
for name, model in models.items():
    scores = cross_val_score(
        model, X_train_scaled, y_train,
        cv=5, scoring='r2'
    )
    results[name] = scores
    print(f"{name:12s}: CV R² = {scores.mean():.4f} (±{scores.std():.4f})")

# 训练所有模型并记录系数
fig, axes = plt.subplots(1, 4, figsize=(16, 4))
for ax, (name, model) in zip(axes, models.items()):
    model.fit(X_train_scaled, y_train)
    ax.stem(range(30), model.coef_)
    ax.set_title(f'{name}', fontsize=13)
    ax.set_xlabel('特征序号')
    ax.set_ylabel('系数')
    ax.axhline(y=0, color='black', linestyle='--', alpha=0.3)

plt.suptitle('四种模型的系数对比', fontsize=15, y=1.02)
plt.tight_layout()
plt.savefig('model_comparison.png', dpi=150, bbox_inches='tight')
plt.show()

OLS         : CV R² = -2.0364 (±1.5025)
Ridge       : CV R² = -0.5135 (±0.5314)
Lasso       : CV R² = -1.4404 (±1.3278)
ElasticNet  : CV R² = -0.3716 (±0.4478)

从这个对比图中你能清晰看到：

  • OLS：系数的幅度很大且方向不稳定，许多无信息特征被赋予了不应有的权重
  • Ridge：所有系数都被压缩，但没有任何一个精确为零，整体轮廓平滑
  • Lasso：大部分系数直接归零，只保留了极少数非零系数——自动特征选择
  • ElasticNet：介于二者之间，部分归零、部分压缩

5.7 多项式回归中的正则化——一个直观的过拟合演示

这可能是理解正则化最漂亮的实验：

np.random.seed(42)

# 生成正弦数据
X_sin = np.linspace(0, 1, 20).reshape(-1, 1)
y_sin = np.sin(2 * np.pi * X_sin).ravel() + np.random.normal(0, 0.2, 20)

X_test_sin = np.linspace(0, 1, 200).reshape(-1, 1)
y_test_sin = np.sin(2 * np.pi * X_test_sin).ravel()

# 构造多项式特征（degree=15）
poly = PolynomialFeatures(degree=15, include_bias=False)
X_poly = poly.fit_transform(X_sin)
X_test_poly = poly.transform(X_test_sin)

# 标准化
scaler = StandardScaler()
X_poly_scaled = scaler.fit_transform(X_poly)
X_test_poly_scaled = scaler.transform(X_test_poly)

# 三个模型
fig, axes = plt.subplots(1, 3, figsize=(18, 5))

for ax, (name, model) in zip(axes, [
    ('OLS (过拟合)', LinearRegression()),
    ('Ridge (α=0.1)', Ridge(alpha=0.1)),
    ('Ridge (α=10)', Ridge(alpha=10))
]):
    model.fit(X_poly_scaled, y_sin)
    y_pred = model.predict(X_test_poly_scaled)
    
    ax.scatter(X_sin, y_sin, color='black', s=40, zorder=5, label='训练数据')
    ax.plot(X_test_sin, y_test_sin, '--', color='gray', lw=2, label='真实函数')
    ax.plot(X_test_sin, y_pred, color='red', lw=2, label=f'模型预测\n训练 R²={model.score(X_poly_scaled, y_sin):.3f}')
    ax.set_title(name, fontsize=14)
    ax.legend(fontsize=10)
    ax.set_xlabel('x')
    ax.set_ylabel('y')

plt.suptitle('多项式回归中的正则化效果', fontsize=16, y=1.02)
plt.tight_layout()
plt.savefig('polynomial_ridge.png', dpi=150, bbox_inches='tight')
plt.show()

OLS 的曲线为了穿过每一个训练点而剧烈震荡（方差巨大），Ridge α=0.1 的曲线平滑了很多，Ridge α=10 的曲线几乎变成了直线。三种情况恰好对应了三张图：过拟合→恰到好处→欠拟合。

六、标准化不可省略

一个经常被忽视但极其重要的细节：岭回归对特征的尺度敏感。

因为 L2 惩罚

λ∑j=1pβj2\lambda\sum_{j=1}^{p}\beta_j^2 λj=1∑p​βj2​

对所有系数一视同仁，如果某个特征的原始值范围是 [0,1][0, 1][0,1]，另一个是 [0,10000][0, 10000][0,10000]，模型会对第二个特征施加不成比例的大惩罚——不是因为它的信号弱，而仅仅因为它的数值大。

因此在使用岭回归之前，必须对所有特征做标准化（均值为 0，方差为 1）。scikit-learn 的 StandardScaler 可以做这件事，或者直接把 Ridge 包装在 Pipeline 中：

from sklearn.pipeline import make_pipeline

pipe = make_pipeline(
    StandardScaler(),
    Ridge(alpha=1.0)
)
pipe.fit(X_train, y_train)

这样你永远不用担心忘记标准化。

七、为什么不叫 L2 回归而叫"岭"回归？

"Ridge" 这个名字来源于 1970 年 Hoerl 和 Kennard 的论文 Ridge Regression: Biased Estimation for Nonorthogonal Problems。他们观察到，当在参数空间的等高线图上绘制解的轨迹时，从 OLS 解到最优 Ridge 解的路径像一条山脊线（ridge），因此得名。

更有趣的是，在中文文献中曾被译为"脊回归"或"岭回归"，"岭"这个译法最终胜出——可能是因为它的确让人联想到"在山岭中找到一条稳定路径"的画面。

八、总结

岭回归是机器学习中最简洁优雅的正则化方法之一。它用一行公式

β^Ridge=(XTX+λI)−1XTy\hat{\beta}_{\text{Ridge}} = (X^T X + \lambda I)^{-1} X^T y β^​Ridge​=(XTX+λI)−1XTy

解决了线性回归在高维、共线性数据上的崩溃问题。它的核心哲学是——宁愿做一个轻微有偏但稳定可靠的预测，也不做一个无偏但方差大到不可用的估计。这句话在机器学习中反复回响，本质上也是所有正则化方法的共同信念。

何时用 Ridge：

  • 特征之间存在多重共线性（VIF 高）
  • 样本数少于特征数（n<pn < pn<p）
  • 你不想丢弃任何特征，只想让它们的系数更保守
  • 你需要一个简单、可解释、有闭式解的基线模型

何时不用 Ridge：

  • 你需要稀疏解（特征选择）——用 Lasso
  • 你两者都想要——用 ElasticNet
  • 你的特征之间几乎没有相关性，且样本远多于特征——OLS 可能就够了
