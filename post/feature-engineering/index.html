特征工程：决定模型上限的隐形艺术家

2026-04-28 21:04 58 min read 人阅读

机器学习特征工程完全指南：数据是原石，特征工程是雕琢

引言：数据与算法的真正桥梁

在机器学习领域流传着一句经典名言：

“数据和特征决定了机器学习的上限，而模型和算法只是逼近这个上限。”

这意味着，无论你使用多么先进的深度学习模型，如果输入的特征质量低下，模型的性能都将受到根本性的限制。特征工程（Feature Engineering）就是将原始数据转化为能够更好表达问题本质的特征的过程，它是机器学习工作流中最耗时、最考验经验，但也最可能带来性能飞跃的环节。

本文将全面梳理特征工程的四大核心模块：数据清洗与预处理、特征变换、特征构建、特征选择，并提供大量可立即使用的 Python 代码片段。

第一章：数据清洗与预处理——地基必须牢固

1.1 缺失值处理

现实数据几乎不可能完美无缺。缺失值的处理直接影响模型效果。

方法一：删除

  • 删除缺失比例过高的特征（例如缺失超过 50%）。
  • 删除包含缺失值的样本（当数据量巨大且缺失比例极低时）。

方法二：填充

  • 数值特征：用均值、中位数、众数填充，或用模型预测缺失值（如 KNN 插补）。
  • 类别特征：用众数填充，或将缺失视为一个新的类别（如“未知”）。

方法三：使用能够处理缺失值的模型

  • 如 XGBoost、LightGBM 可以自动处理缺失值。
import pandas as pd
import numpy as np
from sklearn.impute import SimpleImputer

# 均值填充数值特征
num_imputer = SimpleImputer(strategy='mean')
df[['age', 'salary']] = num_imputer.fit_transform(df[['age', 'salary']])

# 众数填充分类特征
cat_imputer = SimpleImputer(strategy='most_frequent')
df[['city']] = cat_imputer.fit_transform(df[['city']])

1.2 异常值处理

异常值可能严重干扰模型，尤其是基于距离或梯度的模型（如线性回归、K-Means）。

检测方法：

  • 箱线图法：值小于 Q1−1.5×IQRQ_1 - 1.5 \times IQRQ1​−1.5×IQR 或大于 Q3+1.5×IQRQ_3 + 1.5 \times IQRQ3​+1.5×IQR 视为异常。
  • Z-Score 法：当 ∣Z∣>3|Z| > 3∣Z∣>3 时通常视为异常（假设正态分布）。
  • Isolation Forest：无监督算法专门检测异常点。

处理策略：

  • 截断：将异常值替换为边界值（如第1百分位和第99百分位）。
  • 分箱：将连续值离散化为区间，减轻异常值影响。
  • 保留：如果异常值代表真实极端事件（如欺诈、灾害），则需要保留。
# 截断异常值
q1 = df['price'].quantile(0.01)
q99 = df['price'].quantile(0.99)
df['price'] = df['price'].clip(lower=q1, upper=q99)

1.3 数据标准化与归一化

许多机器学习模型假设特征在相似的尺度上。尺度差异大的特征会导致梯度下降收敛缓慢，也会让基于距离的模型（KNN、SVM）被大尺度特征主导。

  • 标准化（Standardization）：x′=x−μσx' = \frac{x - \mu}{\sigma}x′=σx−μ​，使特征均值为 0，标准差为 1。适用于假设数据服从正态分布的模型（如线性回归、逻辑回归、LDA）。
  • 归一化（Min-Max Scaling）：x′=x−xminxmax−xminx' = \frac{x - x_{min}}{x_{max} - x_{min}}x′=xmax​−xmin​x−xmin​​，缩放到 [0,1][0, 1][0,1] 区间。适用于需要固定范围输入的模型（如神经网络使用 Sigmoid 激活时）。
from sklearn.preprocessing import StandardScaler, MinMaxScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

第二章：特征变换——让数据适配模型

2.1 非线性变换

许多模型假设特征与目标之间存在线性关系，但现实往往并非如此。通过非线性变换可以“拉伸”或“压缩”数据分布，使其更符合模型假设。

常用变换：

  • 对数变换 log⁡(x+1)\log(x+1)log(x+1)：适用于右偏严重的正值数据（如收入、金额），能减小极值的影响并使其接近正态分布。
  • 平方根变换：效果与对数变换类似，但对 0 值更友好。
  • Box-Cox 变换：一种广义的幂变换，能自动调整参数以最大化正态性，仅适用于正数。
  • Yeo-Johnson 变换：Box-Cox 的扩展，支持零和负数。
from sklearn.preprocessing import PowerTransformer

# Box-Cox 变换（需数据为正）
pt_boxcox = PowerTransformer(method='box-cox')
X_boxcox = pt_boxcox.fit_transform(X_positive)

# Yeo-Johnson 变换（支持任意实数）
pt_yeo = PowerTransformer(method='yeo-johnson')
X_yeo = pt_yeo.fit_transform(X)

2.2 离散化（分箱）

将连续特征转换为离散的“箱子”，可以提高模型的非线性表达能力，同时增加稳健性，减少异常值干扰。

  • 等宽分箱：将取值范围均匀划分。
  • 等频分箱：使每个箱包含大致相同数量的样本。
  • 基于决策树的分箱：利用目标变量的信息有监督地进行最优分箱。
from sklearn.preprocessing import KBinsDiscretizer

# 等频分箱，分为 5 个箱子，输出为整数编码
est = KBinsDiscretizer(n_bins=5, encode='ordinal', strategy='quantile')
X_binned = est.fit_transform(X)

2.3 类别特征编码

模型一般要求输入是数值，因此必须将类别特征转化为数值。

编码方法 原理 适用场景
One-Hot 编码 每个类别生成一个新 0/1 特征 类别数较少（<10），无顺序关系
标签编码 将类别映射为整数 有序类别（如：低、中、高）
频数编码 用类别出现频数/频率替代 高基数特征（>100 个类别）
目标编码 用目标变量的均值（或其他统计量）替代 高基数特征且有监督学习任务
嵌入编码 用神经网络将高维稀疏类别映射到低维稠密向量 深度学习模型，超高基数 
import pandas as pd
from sklearn.preprocessing import OneHotEncoder, LabelEncoder

# One-Hot 编码（常用）
ohe = OneHotEncoder(sparse_output=False, handle_unknown='ignore')
X_ohe = ohe.fit_transform(df[['city']])

# 频数编码
freq_map = df['city'].value_counts().to_dict()
df['city_freq'] = df['city'].map(freq_map)

2.4 文本特征提取

  • 词袋模型和 TF-IDF：将文本转化为词频或加权词频向量。
  • 预训练词嵌入：Word2Vec、GloVe、BERT 等。
  • 文本统计特征：字符数、单词数、平均词长、大写字母比例、标点频率等。

2.5 时间/日期特征

从时间戳中可以提取出丰富的周期性特征：

  • 年、月、日、时、分、秒
  • 星期几（1-7）、是否周末、是否节假日
  • 季度、月份所属的季节
  • 距某一关键日期的天数
df['date'] = pd.to_datetime(df['date'])
df['year'] = df['date'].dt.year
df['month'] = df['date'].dt.month
df['day_of_week'] = df['date'].dt.dayofweek
df['is_weekend'] = (df['day_of_week'] >= 5).astype(int)

第三章：特征构建——创造新特征的艺术

3.1 组合特征与交互特征

两个特征组合在一起可能产生远大于单独相加的信息量。

  • 加、减、乘、除：例如 人均消费 = 总消费 / 人数。
  • 多项式特征：生成特征之间的二次或更高次交互项。
  • 特征交叉：将两个或多个类别特征拼接成一个新特征，例如将“城市”和“产品类别”组合成“城市_产品”。
from sklearn.preprocessing import PolynomialFeatures

# 生成最高 2 次的交互特征
poly = PolynomialFeatures(degree=2, interaction_only=False, include_bias=False)
X_poly = poly.fit_transform(X[['height', 'width']])

3.2 统计特征

在表格型数据竞赛中，统计特征是最有力的武器之一。可以从多个相关数据源（如历史记录、分组聚合）中计算统计量。

  • 分组聚合：对用户按时间窗口计算平均值、最大值、总和、标准差等。
  • 计数特征：某个事件发生的次数。
  • 比率特征：两个统计量相除。
# 按用户分组，计算消费金额的均值和总和
user_stats = df.groupby('user_id').agg(
    mean_spending=('amount', 'mean'),
    total_spending=('amount', 'sum'),
    purchase_count=('amount', 'count')
).reset_index()

3.3 业务领域知识特征

这是机器学习中最难以自动化，也最具价值的部分。深入理解业务，能创造出决定性的“黄金特征”。

  • 金融风控：收入与负债比率、近期交易频率激增度。
  • 零售：复购率、客户生命周期长度、促销敏感度。
  • 交通：早晚高峰时段标识、相邻路段速度差。

3.4 目标编码与降维作为特征

  • 目标编码：如前所述，对分类变量用目标均值编码（需配合平滑和交叉验证防止过拟合）。
  • PCA / t-SNE / UMAP：对原始特征进行降维后得到的新特征，可作为模型输入。
  • 聚类标签：先用 K-Means 等算法对数据进行聚类，将聚类标签作为新特征。
  • 模型集成特征：用另一个模型（如 GBDT）输出决策树叶子节点的 one-hot 编码作为特征。

第四章：特征选择——提炼特征中的精华

4.1 过滤法

独立于模型，通过统计指标快速筛选特征。

  • 方差阈值法：剔除方差极低（几乎为常数）的特征。
  • 相关系数法：保留与目标变量相关性高的特征，剔除特征间高度相关的冗余特征。
  • 卡方检验（分类任务）：衡量分类特征与目标之间的独立性。
  • 互信息法：衡量特征与目标之间的非线性依赖关系。
from sklearn.feature_selection import VarianceThreshold, SelectKBest, mutual_info_regression

# 删除方差低于 0.01 的特征
selector = VarianceThreshold(threshold=0.01)
X_var = selector.fit_transform(X)

# 选择与目标互信息最高的 10 个特征
selector_mi = SelectKBest(mutual_info_regression, k=10)
X_mi = selector_mi.fit_transform(X, y)

4.2 包装法

使用目标模型来评估特征子集，通过搜索得到最优组合。

  • 递归特征消除（RFE）：用模型反复训练，每次删除最不重要的特征，直到剩余指定数量。
  • 前向搜索/后向搜索：逐个添加或删除特征，根据模型性能变化决定取舍。
from sklearn.feature_selection import RFE
from sklearn.linear_model import LogisticRegression

# 使用逻辑回归作为基模型，选择最佳 10 个特征
rfe = RFE(estimator=LogisticRegression(), n_features_to_select=10)
X_rfe = rfe.fit_transform(X, y)

4.3 嵌入法

模型在训练过程中自动完成特征选择。

  • L1 正则化（Lasso）：将不重要特征的系数压缩为零。
  • 树模型的重要性：随机森林、XGBoost 等可输出特征被分裂的累计增益或次数，直观反映特征重要性。
  • Permutation Importance（排列重要性）：打乱某特征的值，观察模型性能下降程度，以此评估重要性。
from sklearn.linear_model import LassoCV

# Lasso 自动特征选择
lasso = LassoCV(cv=5, random_state=42)
lasso.fit(X, y)
selected_features = X.columns[lasso.coef_ != 0]

第五章：特征工程工作流与最佳实践

5.1 特征工程不是一次性的

  • 持续迭代：新特征可能从业务反馈、数据变化中不断涌现。
  • 特征存储：建立 Feature Store（如 Feast）来统一管理特征定义和计算，避免线上线下不一致。

5.2 防止数据泄露

特征工程中最高发的事故就是数据泄露：在构建特征时无意识地使用了未来的信息、或在训练之前对整个数据集进行了标准化。

  • 标准化/归一化：必须先划分训练/测试集，仅用训练集数据拟合 Scaler，再分别转换。
  • 目标编码：必须在交叉验证折内计算目标均值，不能全局计算。
  • 分组聚合：使用 groupby 时，必须确保聚合所用的切片不包含测试集样本。

5.3 自动化特征工程

  • FeatureTools：能自动进行深度特征综合（Deep Feature Synthesis），组合基本特征生成复杂特征。
  • AutoML 框架：如 H2O.ai、AutoGluon 等内置了自动特征工程和特征选择。

第六章：总结：特征工程的至高心法

  1. 懂业务：最好的特征往往来自于对场景的深刻理解。
  2. 多尝试：特征工程不存在万能公式，需要大量实验来验证有效性。
  3. 防泄漏：任何不经意的信息泄露都可能导致模型上线失效。
  4. 可解释：生成的特征最好有明确的业务含义，方便日后的诊断和维护。
  5. 先简单后复杂：从基础的聚合、比率特征开始，逐步尝试复杂的嵌入或模型集成特征。

特征工程是机器学习中最有“人味”的部分，它决定了数据的表达方式，也决定了模型能看到的世界。把数据锤炼成特征，你就是那道连接现实世界与算法世界的桥梁。
