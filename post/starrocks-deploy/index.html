StarRocks 存算一体集群部署实战：Linux 环境 3FE + 4BE 从 0 到可用

2026-05-16 21:59 46 min read 人阅读

本文是 StarRocks 存算一体 shared-nothing 模式 的手动部署方案。该模式下 BE 同时负责数据存储和 SQL 计算，数据落在 BE 本地磁盘，不使用 CN，不配置对象存储或 HDFS 作为共享数据层。如果实际要部署存算分离模式，应使用 FE + CN + 远程存储的方案。

一、部署目标

本文档用于在 Linux 环境手动部署一个 StarRocks 集群：

  • 3 个 FE：1 个 Leader FE + 2 个 Follower FE，用于元数据高可用。
  • 4 个 BE：同时负责数据存储和 SQL 计算。
  • 部署模式：存算一体 shared-nothing。
  • 存储方式：数据存储在 BE 本地磁盘。
  • 不涉及组件：CN、对象存储、HDFS 共享数据层。
  • 访问协议：MySQL 协议，默认端口 9030。
  • FE 元数据目录：/data/starrocks/meta。
  • BE 数据目录：/data/starrocks/storage。
  • StarRocks 安装目录：/opt/starrocks。

二、架构说明

存算一体模式是 StarRocks 最经典的部署形态，也叫 shared-nothing 模式。它的核心特点是：

  • FE 负责管控和调度：管理元数据、接收 SQL、生成执行计划、调度查询任务。
  • BE 负责存储和计算：每个 BE 都有本地数据目录，既保存 Tablet 副本，也执行 Scan、Join、Aggregation 等计算任务。
  • 数据不依赖共享存储：不需要 S3、MinIO、HDFS 等远程共享存储作为主数据层。
  • 副本保证可靠性：表数据会按照副本数分布在多个 BE 上，常见生产配置为 3 副本。
  • 水平扩展简单：计算和存储能力随 BE 数量一起扩展，适合实时数仓、报表分析、明细查询和高并发 OLAP 场景。

本文的目标拓扑如下：

MySQL Client / BI Tool
        |
        v
+-----------------------------+
| FE 集群：3 个 FE             |
| 1 Leader + 2 Follower        |
+-----------------------------+
        |
        v
+-----------------------------+
| BE 集群：4 个 BE             |
| 本地存储 + SQL 并行计算       |
+-----------------------------+

三、节点规划

节点角色 主机名 IP 示例 组件 说明
FE sr-fe-01 192.168.10.11 FE Leader 首个启动的 FE，后续自动参与选主
FE sr-fe-02 192.168.10.12 FE Follower 元数据副本，参与选主
FE sr-fe-03 192.168.10.13 FE Follower 元数据副本，参与选主
BE sr-be-01 192.168.10.21 BE 数据存储与计算
BE sr-be-02 192.168.10.22 BE 数据存储与计算
BE sr-be-03 192.168.10.23 BE 数据存储与计算
BE sr-be-04 192.168.10.24 BE 数据存储与计算

实际部署时，将文档中的 IP 替换为真实服务器 IP。

四、端口规划

4.1 FE 端口

端口 配置项 用途
8030 http_port FE HTTP 服务端口
9010 edit_log_port FE 元数据日志同步端口
9020 rpc_port FE Thrift RPC 端口
9030 query_port MySQL 协议查询端口
6090 cloud_native_meta_port 云原生元数据 RPC 端口，存算一体部署通常不需要主动配置

4.2 BE 端口

端口 配置项 用途
8040 be_http_port BE HTTP 服务端口
8060 brpc_port BE bRPC 端口
9050 heartbeat_service_port BE 心跳端口，FE 通过该端口管理 BE
9060 be_port BE Thrift 服务端口
9070 starlet_port BE agent 服务端口

生产环境需要在防火墙、安全组或 ACL 中放通上述端口。

五、系统准备

以下操作建议在所有 FE 和 BE 服务器上执行。

5.1 创建用户和目录

sudo useradd -m -s /bin/bash starrocks
sudo mkdir -p /opt/starrocks
sudo mkdir -p /data/starrocks/meta
sudo mkdir -p /data/starrocks/storage
sudo chown -R starrocks:starrocks /opt/starrocks /data/starrocks

FE 节点需要 /data/starrocks/meta，BE 节点需要 /data/starrocks/storage。如果 FE 和 BE 分开部署，可以只创建本节点需要的目录。

5.2 配置主机名解析

在所有节点的 /etc/hosts 中加入：

192.168.10.11 sr-fe-01
192.168.10.12 sr-fe-02
192.168.10.13 sr-fe-03
192.168.10.21 sr-be-01
192.168.10.22 sr-be-02
192.168.10.23 sr-be-03
192.168.10.24 sr-be-04

如果使用 IP 部署，也建议维护主机名解析，后续运维更清晰。

5.3 安装 JDK

StarRocks 依赖 JDK，不支持只安装 JRE。

  • StarRocks v3.3、v3.4：JDK 11 或更高版本。
  • StarRocks v3.5 及更高版本：JDK 17 或更高版本。

以 JDK 17 为例：

java -version
echo $JAVA_HOME

如果未配置 JAVA_HOME，可以在 /etc/profile 中加入：

export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$PATH:$JAVA_HOME/bin

然后执行：

source /etc/profile
java -version

5.4 检查 CPU 指令集

StarRocks 的向量化执行强依赖 CPU 能力，生产环境建议使用支持 AVX2 的 x86 机器。

cat /proc/cpuinfo | grep avx2

有输出表示 CPU 支持 AVX2。

5.5 配置系统参数

sudo tee -a /etc/sysctl.conf <<'EOF'
vm.overcommit_memory=1
vm.swappiness=0
net.ipv4.tcp_abort_on_overflow=1
net.core.somaxconn=1024
EOF

sudo sysctl -p

5.6 配置 limits

sudo tee -a /etc/security/limits.conf <<'EOF'
* soft nproc 65535
* hard nproc 65535
* soft nofile 655350
* hard nofile 655350
* soft stack unlimited
* hard stack unlimited
* soft memlock unlimited
* hard memlock unlimited
EOF

重新登录后验证：

ulimit -n
ulimit -u

5.7 关闭 Swap

sudo swapoff -a
free -m

同时检查 /etc/fstab，注释掉 swap 挂载项，避免重启后再次启用。

5.8 配置透明大页

echo madvise | sudo tee /sys/kernel/mm/transparent_hugepage/enabled
echo madvise | sudo tee /sys/kernel/mm/transparent_hugepage/defrag

可以加入 /etc/rc.d/rc.local 或 systemd 启动脚本中持久化。

5.9 时间同步

所有节点必须保持时间同步，建议使用 chronyd：

sudo systemctl enable chronyd
sudo systemctl start chronyd
timedatectl

5.10 检查端口占用

FE 节点执行：

netstat -tunlp | grep -E '8030|9010|9020|9030|6090'

BE 节点执行：

netstat -tunlp | grep -E '8040|8060|9050|9060|9070'

如果端口被占用，需要在 fe.conf 或 be.conf 中调整对应配置。

六、准备安装包

在一台运维机或任意节点下载 StarRocks 二进制包。以 x86_64 Linux 为例：

cd /opt/starrocks

# 根据实际版本和操作系统替换 VERSION 与 OS。
# OS 通常为 centos 或 ubuntu。
VERSION=3.5.0
OS=centos

wget https://releases.starrocks.io/starrocks/StarRocks-${VERSION}-${OS}-amd64.tar.gz
tar -xzvf StarRocks-${VERSION}-${OS}-amd64.tar.gz

解压后通常包含：

fe/
be/
apache_hdfs_broker/
LICENSE.txt
NOTICE.txt

分发文件：

  • 将 fe/ 分发到 3 台 FE 节点的 /opt/starrocks/。
  • 将 be/ 分发到 4 台 BE 节点的 /opt/starrocks/。

示例：

scp -r fe starrocks@sr-fe-01:/opt/starrocks/
scp -r fe starrocks@sr-fe-02:/opt/starrocks/
scp -r fe starrocks@sr-fe-03:/opt/starrocks/

scp -r be starrocks@sr-be-01:/opt/starrocks/
scp -r be starrocks@sr-be-02:/opt/starrocks/
scp -r be starrocks@sr-be-03:/opt/starrocks/
scp -r be starrocks@sr-be-04:/opt/starrocks/

七、配置 FE

在 3 台 FE 节点上修改 /opt/starrocks/fe/conf/fe.conf。

7.1 sr-fe-01

meta_dir = /data/starrocks/meta

http_port = 8030
rpc_port = 9020
query_port = 9030
edit_log_port = 9010

priority_networks = 192.168.10.11/32

7.2 sr-fe-02

meta_dir = /data/starrocks/meta

http_port = 8030
rpc_port = 9020
query_port = 9030
edit_log_port = 9010

priority_networks = 192.168.10.12/32

7.3 sr-fe-03

meta_dir = /data/starrocks/meta

http_port = 8030
rpc_port = 9020
query_port = 9030
edit_log_port = 9010

priority_networks = 192.168.10.13/32

priority_networks 用于指定服务绑定/上报的网络地址。多网卡机器必须配置正确，否则节点之间可能无法互相识别。

八、启动 Leader FE

在 sr-fe-01 上执行：

su - starrocks
cd /opt/starrocks
./fe/bin/start_fe.sh --daemon

检查日志：

grep -i "thrift server started" /opt/starrocks/fe/log/fe.log

登录 FE：

mysql -h 192.168.10.11 -P9030 -uroot

初始 root 用户默认无密码。进入 MySQL 客户端后检查 FE：

SHOW PROC '/frontends'\G

能看到 sr-fe-01 或 192.168.10.11，且 Alive: true、Role: LEADER，表示首个 FE 启动成功。

九、添加并启动 Follower FE

9.1 在 Leader FE 中添加 Follower

连接 sr-fe-01：

mysql -h 192.168.10.11 -P9030 -uroot

执行：

ALTER SYSTEM ADD FOLLOWER "192.168.10.12:9010";
ALTER SYSTEM ADD FOLLOWER "192.168.10.13:9010";

9.2 启动 sr-fe-02

su - starrocks
cd /opt/starrocks
./fe/bin/start_fe.sh --helper 192.168.10.11:9010 --daemon

9.3 启动 sr-fe-03

su - starrocks
cd /opt/starrocks
./fe/bin/start_fe.sh --helper 192.168.10.11:9010 --daemon

9.4 验证 FE 集群

在任意 FE 上连接：

mysql -h 192.168.10.11 -P9030 -uroot

执行：

SHOW PROC '/frontends'\G

期望结果：

  • 共 3 个 FE。
  • 1 个 LEADER，2 个 FOLLOWER。
  • 3 个 FE 的 Alive 都为 true。
  • 3 个 FE 的 Join 都为 true。

十、配置 BE

在 4 台 BE 节点上修改 /opt/starrocks/be/conf/be.conf。

10.1 sr-be-01

storage_root_path = /data/starrocks/storage

be_port = 9060
be_http_port = 8040
heartbeat_service_port = 9050
brpc_port = 8060
starlet_port = 9070

priority_networks = 192.168.10.21/32

10.2 sr-be-02

storage_root_path = /data/starrocks/storage

be_port = 9060
be_http_port = 8040
heartbeat_service_port = 9050
brpc_port = 8060
starlet_port = 9070

priority_networks = 192.168.10.22/32

10.3 sr-be-03

storage_root_path = /data/starrocks/storage

be_port = 9060
be_http_port = 8040
heartbeat_service_port = 9050
brpc_port = 8060
starlet_port = 9070

priority_networks = 192.168.10.23/32

10.4 sr-be-04

storage_root_path = /data/starrocks/storage

be_port = 9060
be_http_port = 8040
heartbeat_service_port = 9050
brpc_port = 8060
starlet_port = 9070

priority_networks = 192.168.10.24/32

如果 BE 有多块数据盘，可以使用分号配置多个目录：

storage_root_path = /data1/starrocks/storage;/data2/starrocks/storage;/data3/starrocks/storage

十一、启动 BE

在 4 台 BE 节点上分别执行：

su - starrocks
cd /opt/starrocks
./be/bin/start_be.sh --daemon

检查日志：

grep -i "heartbeat" /opt/starrocks/be/log/be.INFO

BE 首次启动后会等待 FE 心跳，只有加入集群后才会变为可用。

十二、添加 BE 到集群

连接 Leader FE：

mysql -h 192.168.10.11 -P9030 -uroot

执行：

ALTER SYSTEM ADD BACKEND "192.168.10.21:9050";
ALTER SYSTEM ADD BACKEND "192.168.10.22:9050";
ALTER SYSTEM ADD BACKEND "192.168.10.23:9050";
ALTER SYSTEM ADD BACKEND "192.168.10.24:9050";

查看 BE 状态：

SHOW PROC '/backends'\G

期望结果：

  • 共 4 个 BE。
  • 每个 BE 的 Alive 为 true。
  • SystemDecommissioned 为 false。
  • ClusterDecommissioned 为 false。
  • TabletNum、容量、CPU、内存信息能够正常展示。

十三、初始化安全配置

首次部署后，建议立刻修改 root 密码：

SET PASSWORD FOR 'root' = PASSWORD('StrongPassword_ChangeMe');

重新登录：

mysql -h 192.168.10.11 -P9030 -uroot -p

创建业务用户示例：

CREATE USER 'app_user'@'%' IDENTIFIED BY 'AppPassword_ChangeMe';
GRANT ALL ON *.* TO 'app_user'@'%';

生产环境不要直接给业务用户 ALL ON *.*，应按数据库、表、Catalog 维度最小授权。

十四、创建测试库表

CREATE DATABASE IF NOT EXISTS test_db;
USE test_db;

CREATE TABLE site_access (
    event_day DATE,
    site_id INT,
    city_code SMALLINT,
    user_name VARCHAR(32),
    pv BIGINT SUM
)
AGGREGATE KEY(event_day, site_id, city_code, user_name)
DISTRIBUTED BY HASH(site_id) BUCKETS 8
PROPERTIES (
    "replication_num" = "3"
);

INSERT INTO site_access VALUES
('2026-05-16', 1, 10, 'alice', 100),
('2026-05-16', 2, 20, 'bob', 200);

SELECT * FROM site_access;

如果插入和查询正常，说明 FE 查询链路、BE 写入链路和副本调度基本可用。

十五、常用运维命令

十六、启停顺序

16.1 启动顺序

建议顺序：

  1. 启动 3 个 FE。
  2. 启动 4 个 BE。
  3. 登录 FE 验证节点状态。

启动 FE：

cd /opt/starrocks
./fe/bin/start_fe.sh --daemon

Follower FE 首次启动才需要 --helper，后续重启通常不需要再加。

启动 BE：

cd /opt/starrocks
./be/bin/start_be.sh --daemon

16.2 停止顺序

建议先停 BE，再停 FE。

停止 BE：

cd /opt/starrocks
./be/bin/stop_be.sh

停止 FE：

cd /opt/starrocks
./fe/bin/stop_fe.sh

十七、常见问题

17.1 FE 启动失败

优先检查：

tail -200 /opt/starrocks/fe/log/fe.log
tail -200 /opt/starrocks/fe/log/fe.warn.log

常见原因：

  • meta_dir 目录不存在或权限不正确。
  • JDK 版本不符合要求。
  • priority_networks 配错，FE 上报了错误 IP。
  • edit_log_port、query_port 等端口被占用。
  • Follower FE 首次启动没有指定正确的 --helper。

17.2 BE 一直不 Alive

优先检查：

tail -200 /opt/starrocks/be/log/be.INFO
tail -200 /opt/starrocks/be/log/be.WARNING

常见原因：

  • BE 没有执行 ALTER SYSTEM ADD BACKEND。
  • 添加 BE 时使用了错误 IP 或心跳端口。
  • storage_root_path 目录不存在或权限不正确。
  • FE 与 BE 之间网络不通。
  • BE 的 priority_networks 配错。

17.3 MySQL 客户端无法连接

检查：

netstat -tunlp | grep 9030
telnet 192.168.10.11 9030

常见原因：

  • FE 未启动。
  • 防火墙或安全组未放通 9030。
  • 连接到了 Follower FE，但当前版本或配置下查询端口未正常提供服务。
  • 密码错误。

17.4 4 个 BE 与 3 副本的关系

StarRocks 默认表副本数通常为 3。4 个 BE 可以正常部署 3 副本表，系统会在 4 个 BE 之间做副本分布和均衡。

不过需要注意：

  • 如果单个 BE 故障，3 副本表仍有可用副本。
  • 如果连续故障多个 BE，可能影响部分 Tablet 可用性。
  • 数据量较大时，应关注磁盘使用率、Tablet 分布和副本均衡情况。

十八、生产建议

  • FE 节点建议使用奇数个，3 个 FE 是常见高可用起点。
  • BE 节点建议使用 SSD 或 NVMe，尤其是主键表、实时写入和高并发分析场景。
  • FE 的 meta_dir 应放在可靠磁盘上，并纳入备份策略。
  • BE 的 storage_root_path 不要放在系统盘。
  • 所有节点必须配置 NTP/Chrony 时间同步。
  • 生产环境必须修改默认 root 密码。
  • 业务连接建议通过负载均衡访问 FE 的 9030 端口。
  • 定期检查 SHOW PROC '/frontends'\G 和 SHOW PROC '/backends'\G。
  • 重要版本升级前，先阅读对应版本 Release Notes 和 Upgrade 文档。
