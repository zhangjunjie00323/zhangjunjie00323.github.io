StarRocks 系统表完全指南：从元数据查询到集群监控

2026-04-29 21:54 69 min read -- 人阅读

玩转 StarRocks 系统库：字段级解析 *statistics*, information_schema, sys

前言

在 StarRocks 中，系统表是我们监控集群健康度、优化查询性能和排查问题的基石。随着版本迭代（特别是 v3.1/v3.2 之后），系统表的体系愈发完善。

本文将为你深度剖析 StarRocks 的四个核心系统库，并对每个系统表的用途和具体字段进行逐项解释。

第一章：_statistics_ 统计信息库

CBO（基于代价的优化器）优化器依赖统计信息生成高效的执行计划。

特别说明：_statistics_ 库下的表结构在 StarRocks v3.2 及以上版本发生了较大重构，以下基于新版本讲解。

1.1 column_statistics

用途：记录列级别的基础统计信息。CBO 优化器依赖此表的数据来估算各种执行计划的代价，从而选择最优的执行路径。此表存储了全量的列级统计信息，包括行数、去重值个数（NDV）、空值个数、最大值和最小值等。

字段名 数据类型 说明
table_id BIGINT 表的唯一ID
partition_id BIGINT 分区的唯一ID
column_name VARCHAR 列的名称
db_id BIGINT 数据库的ID
table_name VARCHAR 表名
partition_name VARCHAR 分区名
row_count BIGINT 该分片的总行数
data_size BIGINT 该分片的数据总大小（字节）
ndv BIGINT Number of Distinct Values，该列不同值的估计个数
null_count BIGINT 该列中 NULL 值的个数
max VARCHAR 该列的最大值
min VARCHAR 该列的最小值

1.2 histogram_statistics

用途：记录列的直方图统计信息。当数据存在严重倾斜或高频值时，仅靠 column_statistics 中的基础信息会产生估算偏差。直方图通过分桶记录数据分布，能帮助优化器更准确地估算过滤条件下的行数，从而选择更优的 Join 顺序和 Join 类型。

字段名 数据类型 说明
table_id BIGINT 表的唯一ID
column_name VARCHAR 列名
db_id BIGINT 数据库的ID
table_name VARCHAR 表名
buckets VARCHAR 直方图的桶信息（JSON 格式）
mcv VARCHAR Most Common Value，高频值及其频次
update_time DATETIME 统计信息的最新更新时间

第二章：information_schema 元数据库

这是兼容 MySQL 协议的元数据库，使用最频繁。它提供了数据库、表、列、分区等核心对象的元数据查询能力。

2.1 tables

用途：记录数据库中所有表的元数据信息。可以查询到表的类型（BASE TABLE 或 VIEW）、存储引擎、近似行数、数据大小、创建时间和更新时间等。常用于表资产盘点、存储容量分析。

字段名 数据类型 说明
TABLE_CATALOG VARCHAR(512) 目录名（默认 def）
TABLE_SCHEMA VARCHAR(64) 数据库名
TABLE_NAME VARCHAR(64) 表名
TABLE_TYPE VARCHAR(64) 表类型（BASE TABLE / VIEW）
ENGINE VARCHAR(64) 存储引擎（默认 StarRocks）
VERSION BIGINT 版本号
ROW_FORMAT VARCHAR(10) 行格式（Fixed / Dynamic）
TABLE_ROWS BIGINT 表的近似行数
AVG_ROW_LENGTH BIGINT 平均行长度（字节）
DATA_LENGTH BIGINT 数据文件大小（字节）
MAX_DATA_LENGTH BIGINT 最大数据容量
INDEX_LENGTH BIGINT 索引文件大小（字节）
DATA_FREE BIGINT 碎片字节数
AUTO_INCREMENT BIGINT 自增ID值
CREATE_TIME DATETIME 创建时间
UPDATE_TIME DATETIME 最近更新时间
CHECK_TIME DATETIME 最近检查表的时间
TABLE_COLLATION VARCHAR(32) 表默认排序规则
CHECKSUM BIGINT 校验和
CREATE_OPTIONS VARCHAR(255) 创建表时的额外选项
TABLE_COMMENT TEXT 表注释

2.2 columns

用途：记录数据库中所有列的元数据信息。可以查询到每个表的列名、数据类型、字符长度、数值精度、小数位数、是否可空、默认值、键类型（主键/唯一键等）以及列注释。这是字段级血缘分析和数据字典构建的核心数据源。

字段名 数据类型 说明
TABLE_CATALOG VARCHAR(512) 目录名
TABLE_SCHEMA VARCHAR(64) 数据库名
TABLE_NAME VARCHAR(64) 表名
COLUMN_NAME VARCHAR(64) 列名
ORDINAL_POSITION BIGINT 列在表中的顺序位置（从1开始）
COLUMN_DEFAULT TEXT 列的默认值
IS_NULLABLE VARCHAR(3) 是否可为空（YES/NO）
DATA_TYPE VARCHAR(64) 数据类型
CHARACTER_MAXIMUM_LENGTH BIGINT 字符类型的最大长度
CHARACTER_OCTET_LENGTH BIGINT 字符类型的最大字节数
NUMERIC_PRECISION BIGINT 数值类型的精度（总位数）
NUMERIC_SCALE BIGINT 数值类型的小数位数
DATETIME_PRECISION BIGINT 日期时间类型的精度
CHARACTER_SET_NAME VARCHAR(32) 字符集名称
COLLATION_NAME VARCHAR(32) 排序规则名称
COLUMN_TYPE MEDIUMTEXT 完整的列类型定义（如 varchar(10)）
COLUMN_KEY VARCHAR(3) 键类型（PRI主键/UNI唯一索引/MUL其他）
EXTRA VARCHAR(27) 额外信息（如 auto_increment）
PRIVILEGES VARCHAR(154) 该列的权限
COLUMN_COMMENT TEXT 列注释

2.3 partitions

用途：记录所有分区的详细信息。可以查询到每个表的分区名称、分区方法（Range/List等）、分区表达式、分区的行数和数据大小。是管理分区生命周期、排查数据倾斜问题的重要入口。

字段名 数据类型 说明
TABLE_CATALOG VARCHAR(512) 目录名
TABLE_SCHEMA VARCHAR(64) 数据库名
TABLE_NAME VARCHAR(64) 表名
PARTITION_NAME VARCHAR(64) 分区名
SUBPARTITION_NAME VARCHAR(64) 子分区名
PARTITION_ORDINAL_POSITION BIGINT 分区顺序位置
PARTITION_METHOD VARCHAR(18) 分区方法（RANGE/LIST等）
PARTITION_EXPRESSION LONGTEXT 分区表达式
PARTITION_DESCRIPTION LONGTEXT 分区描述（如分区键值的范围）
TABLE_ROWS BIGINT 该分区的近似行数
AVG_ROW_LENGTH BIGINT 平均行长度（字节）
DATA_LENGTH BIGINT 数据文件大小（字节）
MAX_DATA_LENGTH BIGINT 最大数据容量
INDEX_LENGTH BIGINT 索引文件大小（字节）
CREATE_TIME DATETIME 分区创建时间
UPDATE_TIME DATETIME 分区最近更新时间

2.4 loads

用途：记录所有导入作业的详细信息和执行状态。通过此表可以追踪 Broker Load、Insert 等各类导入任务的状态（PENDING/LOADING/FINISHED/CANCELLED）、进度、开始和结束时间以及执行的 SQL 语句。是排查数据导入故障的核心监控表。

字段名 数据类型 说明
JOB_ID BIGINT 导入作业ID
LABEL VARCHAR(128) 导入作业的标签（用户指定的唯一标识）
STATE VARCHAR(64) 作业状态（PENDING/LOADING/FINISHED/CANCELLED）
PROGRESS VARCHAR(64) 作业进度描述
TYPE VARCHAR(64) 导入类型（BROKER/INSERT/ROUTINE LOAD等）
PRIORITY VARCHAR(32) 优先级
ETL_INFO VARCHAR(1024) ETL信息
TASK_INFO VARCHAR(1024) 任务详细信息
CREATE_TIME DATETIME 导入作业创建时间
ETL_START_TIME DATETIME ETL阶段开始时间
ETL_FINISH_TIME DATETIME ETL阶段结束时间
LOAD_START_TIME DATETIME 数据加载阶段开始时间
LOAD_FINISH_TIME DATETIME 数据加载阶段结束时间
TRACKING_SQL VARCHAR(65535) 执行的导入SQL语句

2.5 be_tablets

用途：记录每个 BE（后端）节点上所有 Tablet（数据分片） 的分布和健康状态。通过此表可以查看 Tablet 的大小、行数、版本数、Compaction 状态以及是否为坏副本。是诊断数据分布均衡性、副本健康度和版本积压问题的核心表。

字段名 数据类型 说明
BE_ID BIGINT BE节点的唯一ID
TABLE_ID BIGINT 表的ID
PARTITION_ID BIGINT 分区的ID
TABLET_ID BIGINT Tablet的唯一ID
SCHEMA_HASH BIGINT 表结构的哈希值
TABLET_SIZE BIGINT Tablet的数据大小（字节）
NUM_VERSION BIGINT 该Tablet当前有多少个版本
ROW_COUNT BIGINT 该Tablet的行数
DATA_SIZE BIGINT 数据实际占用空间（字节）
COMPACTION_STATUS VARCHAR(128) Compaction（合并）状态
IS_BAD BOOLEAN 是否为坏副本（true表示副本损坏）
VERSION_COUNT BIGINT 历史版本数量

第三章：sys 系统元数据库

记录 StarRocks 特有的权限关系和对象依赖关系，这些表通常以只读视图（View）的形式提供。

3.1 grants_to_roles

用途：查看授予用户自定义角色的所有权限。清晰列出了哪个角色在哪个数据库的哪个表上拥有什么权限，以及该角色能否将权限转授给他人。是梳理集群内角色权限分布的核心视图。

字段名 数据类型 说明
Grantee VARCHAR 被授权者（角色名）
Object_Catalog VARCHAR 权限对象所在的Catalog
Object_Database VARCHAR 权限对象所在的数据库
Object_Name VARCHAR 权限对象的具体名称
Object_Type VARCHAR 对象类型（TABLE/MATERIALIZED VIEW/DATABASE等）
Privilege_Type VARCHAR 授予的权限类型（SELECT/ALTER/INSERT等）
Is_Grantable VARCHAR 是否允许被授权者将此权限授予他人（YES/NO）

3.2 grants_to_users

用途：查看直接授予用户的所有权限。与 grants_to_roles 的区别在于，此表记录的是直接绑定在用户账号上的权限，而非通过角色继承的权限。

字段名 数据类型 说明
Grantee VARCHAR 被授权者（用户名）
Object_Catalog VARCHAR 权限对象所在的Catalog
Object_Database VARCHAR 权限对象所在的数据库
Object_Name VARCHAR 权限对象的具体名称
Object_Type VARCHAR 对象类型（TABLE/MATERIALIZED VIEW/DATABASE等）
Privilege_Type VARCHAR 授予的权限类型（SELECT/ALTER/INSERT等）
Is_Grantable VARCHAR 是否允许被授权者将此权限授予他人（YES/NO）

3.3 role_edges

用途：查看角色之间的继承和授予关系。可以清晰地看到哪个角色被授予给了哪个用户，或者哪个角色继承了另一个角色的权限。是绘制角色权限继承链的关键视图。

字段名 数据类型 说明
FROM_ROLE VARCHAR 源角色（父角色，其权限被继承）
TO_ROLE VARCHAR 目标角色（子角色，继承源角色的权限）
TO_USER VARCHAR 目标用户（角色被直接授予给此用户）

第四章：starrocks_audit_db__ 审计日志库

用于持久化存储和查询 StarRocks 集群中执行过的所有 SQL 审计日志。

前置条件：需要安装并启用审计日志插件，并执行官方的建表脚本，才能在此库下生成 starrocks_audit_tbl__ 表。

4.1 starrocks_audit_tbl__

用途：记录集群中所有SQL执行的审计日志。涵盖查询和导入等操作。记录了谁在什么时间从哪个IP执行了哪条SQL，以及执行耗时、扫描数据量、返回行数、资源消耗（CPU/内存）等关键指标。是安全审计、性能分析与慢查询排查的核心数据源。

字段名 数据类型 说明
queryId VARCHAR(48) 查询唯一ID
timestamp DATETIME 查询开始时间
queryType VARCHAR(12) 查询类型（query / slow_query）
clientIp VARCHAR(32) 客户端IP地址
user VARCHAR(64) 发起查询的用户名
authorizedUser VARCHAR(64) 认证通过的用户名
resourceGroup VARCHAR(64) 使用的资源组
catalog VARCHAR(32) 查询所在的Catalog名
db VARCHAR(96) 查询所在的数据库名
state VARCHAR(8) 执行状态（OK/ERR/EOF）
errorCode VARCHAR(96) 错误码（成功时为空）
queryTime BIGINT 查询总耗时（毫秒）
scanBytes BIGINT 扫描数据量（字节）
scanRows BIGINT 扫描行数
returnRows BIGINT 返回行数
cpuCostNs BIGINT CPU消耗（纳秒）
memCostBytes BIGINT 内存消耗（字节）
stmtId INT 该SQL语句的增量ID
isQuery TINYINT 是否为查询操作（1查询/0导入等）
feIp VARCHAR(32) 处理该语句的FE节点IP地址
stmt STRING 原始SQL语句文本
digest VARCHAR(32) SQL语句的哈希指纹（用于聚类相同的SQL）
planCpuCosts DOUBLE 查询规划阶段消耗的CPU（纳秒）
planMemCosts DOUBLE 查询规划阶段消耗的内存（字节）

总结

系统表是我们监控和管理 StarRocks 的显微镜。_statistics_ 负责查询性能优化，information_schema 负责元数据和基础监控，sys 负责权限梳理，starrocks_audit_db__ 负责 SQL 审计追踪。当遇到性能问题或想了解集群状态时，不妨先去这些系统表里找找线索。
